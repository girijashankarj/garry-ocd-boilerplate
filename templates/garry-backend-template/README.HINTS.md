Notes for generated backend projects:

- Swagger: generate OpenAPI spec from handler schemas with `npm run openapi:generate` and expose Swagger UI at `/docs` for Express by default.
- Linting & formatting: ESLint and Prettier are configured; run `npm run lint` and `npm run format`.
- Tests: Jest is configured with ts-jest and an 80% coverage threshold.
- TypeScript is mandatory in this template.
- Branch names are validated in pre-commit: main, qa, develop, feature/_, fix/_, hotfix/\*, release/DD-MM-YYYY.
- Test structure: every `src` code file should have a matching test file under `tests/src` with the same path (e.g. `src/apis/handlers/get-user/logic/preOperation.ts` -> `tests/src/apis/handlers/get-user/logic/preOperation.test.ts`). Run `npm run test:structure` to validate.
- CI: `.github/workflows/pr-check.yml` runs lint, format check, test structure, and tests on PRs.
- Changesets: This template requires a changeset file to be staged for every commit. Use `npx changeset` to create one.
- Husky: Run `npm run prepare` after `npm install` to enable git hooks.
- The scaffolder runs in interactive mode only. Follow prompts and confirm before generation.
- Database: A simple SQLite + Sequelize example is included. Use `npm run db:sync` to apply models and `npm run db:seed` to add example data. For CI/tests the DB uses in-memory SQLite.
- Project structure: a `src/common` folder exists with `enums`, `types`, `constants`, `interfaces`, `sql`, and `utils` for shared utilities and types.
- Config: `config/theme.json`, `config/client.json`, `config/env.json` are required and must not contain secrets.
- API pattern: for resource APIs follow `src/apis/handlers/<action>/` with:
- `index.ts` (default export handler wrapped by withWrap)
- `schema/requestSchema.json`, `schema/responseSchema.json`, `schema/errorSchema.json`
- `logic/preOperation.ts`, `logic/postOperation.ts`, `logic/businessValidation.ts`, `logic/dbOperation.ts`
- Register routes in `src/apis/routes.ts`.
- Example handlers are scaffolded under `src/apis/handlers/get-user`, `create-user`, `update-user`, `delete-user`.
- Auth: `src/apis/helpers/authMiddleware.ts` protects routes using `API_TOKEN` (Bearer token expected).
- Docs: All documentation is consolidated in `docs/PROJECT-DOCS.md`.
- Badge injection: `scripts/inject-badge.js` can inject CI badge(s) into `README.md` with `node scripts/inject-badge.js --repo owner/repo --badge <name>`.
